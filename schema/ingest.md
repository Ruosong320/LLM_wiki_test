# LLM Wiki 资料摄入规则（Qwen3必遵循）
## 一、摄入触发条件
1.  当raw目录下新增文件（文章、论文、书籍等）时，自动触发摄入流程；
2.  当raw目录下已有文件被修改时（仅人工修改，Qwen3不修改），触发重新摄入；
3.  人工手动触发（通过PyCharm运行脚本，输入触发指令）。

## 二、摄入核心流程
1.  读取raw文件：优先读取Markdown格式，PDF格式先提取文字（若无法提取，提示用户人工转换为Markdown）；
2.  信息提取：提取文件中的核心概念、实体、观点、引用，过滤冗余内容（如广告、无关段落）；
3.  Wiki更新：
    - 若提取的概念/实体已存在于wiki对应目录，仅更新「关联内容」「应用场景」「引用来源」，不覆盖原有核心定义；
    - 若提取的概念/实体不存在，按template.md模板生成新的Wiki页面，放入对应目录；
    - 若提取的观点可形成洞察/综合内容，按模板生成对应页面，或更新已有洞察/综合页；
4.  引用标注：所有从raw提取的内容，必须标注来源（格式：raw/类型/文件名.md），确保可追溯；
5.  交叉链接：自动为新生成/更新的Wiki页面，添加与已有页面的双向链接（如概念页关联相关实体页）。

## 三、禁止行为
1.  禁止修改raw目录下的任何文件；
2.  禁止生成与schema规则冲突的Wiki页面（如命名不规范、模板不符）；
3.  禁止遗漏引用来源，禁止编造信息；
4.  禁止重复生成相同的Wiki页面（若存在重名，按structure.md规则处理）；
5.  禁止擅自修改wiki目录下的人工编辑内容，仅可补充、更新关联内容。

## 四、本地Qwen3专属要求
1.  若读取raw文件失败（如路径错误、文件损坏），输出明确错误提示，告知用户检查文件路径和格式，不擅自创建文件；
2.  生成Wiki页面时，确保编码为UTF-8，避免中文乱码（适配PyCharm编辑）；
3.  摄入完成后，输出详细日志，包含新增/更新的Wiki页面路径、引用来源，便于用户核对。
