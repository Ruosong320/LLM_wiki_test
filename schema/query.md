# LLM Wiki 查询规则（Qwen3必遵循）
## 一、查询触发条件
1.  用户在PyCharm终端输入查询指令（如「查询LLM的定义」「LLM Wiki如何实现自动化」）；
2.  用户通过运行脚本，输入查询参数，触发查询；
3.  人工手动调用脚本中的查询函数，输入查询内容。

## 二、查询响应流程
1.  解析查询意图：明确用户查询的是「概念」「实体」「洞察」还是「综合内容」；
2.  检索Wiki页面：优先从wiki目录中检索对应页面，不直接检索raw目录（仅当Wiki中无相关内容时，才读取raw资料临时补充）；
3.  内容整合：根据查询意图，整合Wiki页面中的核心内容，按「清晰、简洁、有条理」的原则组织语言，不冗余；
4.  引用标注：响应内容中，若涉及从Wiki页面提取的信息，标注对应Wiki页面路径（如 wiki/concepts/LLM.md）；
5.  补充提示：若Wiki中无相关内容，提示用户「当前Wiki未收录该内容，可导入相关raw资料触发摄入更新」。

## 三、响应规范
1.  语言简洁，避免冗长，优先用列表、表格呈现核心内容；
2.  不编造信息，若Wiki中无相关内容，不随意猜测；
3.  若查询内容涉及多个Wiki页面，整合后标注所有关联页面；
4.  响应结束后，提示用户「可在PyCharm中手动编辑Wiki页面补充内容」。

## 四、本地Qwen3专属要求
1.  响应内容需适配PyCharm终端显示，避免过长段落，关键信息用列表呈现；
2.  若检索Wiki页面失败（如页面不存在、路径错误），提示用户检查wiki目录结构，不擅自创建页面；
3.  临时补充raw资料时，仅读取内容，不修改raw文件，且需标注「临时补充自raw/类型/文件名.md」。
